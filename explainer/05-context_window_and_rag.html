The context window is the GPU's entire universe
The model has no memory of past conversations, no access to the internet, no database connection. At the moment of inference, the only thing it can "see" is whatever is inside the context window — a flat sequence of tokens. Everything — instructions, history, retrieved documents, your question — must be packed into that one sequence before any GPU work begins.
What actually happens during prefill with a long context
All ~15,500 tokens (system + history + RAG docs + query) are fed into the GPU simultaneously during prefill. Every token attends to every other token — that's the O(n²) attention cost. The KV vectors for all 15,500 tokens are written into HBM. Then decode begins: each new output token only needs to read those cached KV vectors plus run one forward pass. The expensive part was the prefill — that's why TTFT (time to first token) spikes with long contexts.

The crucial thing the model doesn't know
Once everything is packed into the token sequence, the GPU sees one flat stream of tokens. It doesn't know which tokens came from RAG retrieval vs which were typed by the user vs which were the system prompt. They all get the same treatment — embedded, passed through every layer, attended to equally. The "source" labels are invisible to the model. The only signal is positional ordering and whatever formatting the context builder adds (e.g. wrapping retrieved docs in XML tags that the system prompt instructs the model to treat as reference material).
